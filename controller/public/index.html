SIM-PIPE API Reference

This is the reference documentation for the SIM-PIPE API which is implemented using GraphQL. The API is authenticated and user authentication is managed using keycloak.
The URL for token retrieval is " https://datacloud-auth.euprojects.net/auth/realms/user-authentication/protocol/openid-connect/token".

More documentation on how to query a GraphQL API can be found here.

Some tutorials for GraphQL beginners are also listed below:

  • GraphQL Query Basics
  • GraphQL Queries - Fetching data

Queries

dryRun

Description

Get a dry run by ID

Response

Returns a DryRun!

Arguments
Name Description
dryRunId - String! ID of the dry run to be fetched

Example

Query
query dryRun($dryRunId: String!) {
  dryRun(dryRunId: $dryRunId) {
    id
    createdAt
    argoWorkflow
    status {
      phase
      startedAt
      finishedAt
      progress
      message
      estimatedDuration
    }
    project {
      name
      id
      createdAt
      dryRuns {
        ...DryRunFragment
      }
      workflowTemplates {
        ...WorkflowTemplateFragment
      }
    }
    nodes {
      id
      type
      name
      displayName
      templateName
      phase
      startedAt
      finishedAt
      duration
      progress
      children {
        ...DryRunNodeFragment
      }
    }
    node {
      id
      type
      name
      displayName
      templateName
      phase
      startedAt
      finishedAt
      duration
      progress
      children {
        ...DryRunNodeFragment
      }
    }
  }
}
Variables
{"dryRunId": "abc123"}
Response
{
  "data": {
    "dryRun": {
      "id": "abc123",
      "createdAt": "abc123",
      "argoWorkflow": ArgoWorkflow,
      "status": DryRunStatus,
      "project": Project,
      "nodes": [DryRunNode],
      "node": DryRunNode
    }
  }
}

ArgoWorkflow

Description

ArgoWorkflow is the definition of an executable pipeline in Argo. This definition includes all the details about the specific workflow instance you want to run, including the sequence of steps, inputs, outputs, and other configuration parameters. This is required to create and start dry runs in SIM-PIPE.

PrometheusStringNumber

Description

Prometheus represents number as strings in JSON, because the numbers in JSON are not accurate enough and doesn't support very large numbers. This is because they are represented as IEE 754 floating point numbers, which are not accurate and do not represents numbers above 2^53 accurately. The overhead is minimal as the numbers are represented as strings in JSON anyway. You will need to parse the string to a number in your application.
